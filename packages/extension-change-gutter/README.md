# Change Gutter

在 Yank Note 的 Monaco 编辑器装订线中显示新增、修改和删除的行。

- Git 文件使用 `HEAD` 中的文件内容作为基线，保存后标记仍然保留。
- 非 Git、Git 不可用或加密文件使用 Tab 首次打开时的内容作为基线。
- Git 与 Tab 基线沿用相同的变更语义色；Tab 基线使用点状线和紫色删除三角，便于区分来源。
- 仅在非分享模式、当前使用默认编辑器且文档为可编辑 plain 文件时工作；禁用扩展即可关闭功能。
- 首次打开 Tab 只记录快照；用户编辑、切回 Tab 或窗口重新聚焦后才获取 Git 基线并计算标记。
- 点击装订线可展开当前完整变更块，并可将这一块还原到 HEAD 或 Tab 打开时的内容；还原支持正常撤销。
- 面板中的“清除全部标记”会从当前内容重新记录变更，并停止该 Tab 后续的 Git 检查；关闭 Tab 后恢复自动基线选择。
- 面板中的刷新图标会退出手动基线，强制重新获取 Git 或 Tab 基线并计算标记。
- 面板打开时可按 `Esc` 关闭；无论焦点在编辑器还是面板按钮上都生效。
- “文档历史”按钮直接打开 Yank Note 内置文档历史面板。
- 变更内容默认不自动换行；“自动换行”位于关闭按钮之后，状态只保存在当前运行内存中。横向滚动时，带独立增删配色的 `+`/`-` 标记固定在左侧。

Git 基线通过 Yank Note 已有的管理员 RPC 获取。本地 Git 命令不可用、RPC 无权限或执行失败时，扩展会自动降级到 Tab 基线，不影响编辑。

当前版本不提供暂存、提交或完整 Source Control 面板。

详细设计见 [DESIGN.md](./DESIGN.md)。
